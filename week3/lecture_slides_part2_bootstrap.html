What is Bootstrap?

A Powerful CSS Framework

Bootstrap provides pre-built CSS classes for common web components and layouts.

Getting Started - Add One Line!

<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css" rel="stylesheet">

Why Use Bootstrap?

  • 🚀 Speed: Build responsive sites quickly
  • 📱 Mobile-First: Designed for mobile devices first
  • 🎨 Consistent: Professional look across all browsers
  • 🔧 Customizable: Override with your own CSS
  • 📚 Well-Documented: Extensive documentation and examples

Key Concept: Bootstrap uses classes to style elements
Just add class names to your HTML elements!

Responsive Grid Classes

Different Layouts for Different Screens

Use breakpoint-specific classes to create responsive layouts:

<!-- Stack on mobile, side-by-side on tablets and up --> <div class="row"> <div class="col-12 col-md-6 col-lg-4"> <!-- Full width mobile, half tablet, third desktop --> Content here </div> <div class="col-12 col-md-6 col-lg-4"> Content here </div> <div class="col-12 col-md-12 col-lg-4"> Content here </div> </div>

Grid Breakpoint Classes:

  • .col- - Extra small (all devices)
  • .col-sm- - Small devices (≥576px)
  • .col-md- - Medium devices (≥768px)
  • .col-lg- - Large devices (≥992px)
  • .col-xl- - Extra large (≥1200px)
  • .col-xxl- - XX-Large (≥1400px)

Responsive Bootstrap Features

Mobile-First by Default

Responsive Utilities:

<!-- Hide on mobile, show on desktop --> <div class="d-none d-md-block"> Desktop only content </div> <!-- Show on mobile, hide on desktop --> <div class="d-block d-md-none"> Mobile only content </div> <!-- Responsive spacing --> <div class="p-2 p-md-4 p-lg-5"> Padding changes with screen size </div>

Display Classes:

  • .d-none - Hide element
  • .d-block - Display block
  • .d-inline - Display inline
  • .d-flex - Display flex

Responsive Tables:

<div class="table-responsive"> <table class="table"> <!-- Table content --> </table> </div>

Pro Tip: Always test your Bootstrap sites on different devices!

Bootstrap Best Practices

Do's and Don'ts

✅ DO:

  • Use containers to wrap content
  • Follow the 12-column grid system
  • Use responsive classes (col-sm-, col-md-, etc.)
  • Override Bootstrap with custom CSS when needed
  • Keep Bootstrap updated

❌ DON'T:

  • Modify Bootstrap's source files
  • Use more than 12 columns in a row
  • Forget the viewport meta tag
  • Mix Bootstrap versions
  • Overuse utility classes
