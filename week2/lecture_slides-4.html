The CSS Box Model

Every Element is a Box!

Understanding the box model is fundamental to CSS layout. Every HTML element is surrounded by boxes.

MARGIN
BORDER
PADDING
CONTENT

Your text goes here

The Four Boxes (From Inside Out):

  1. Content: The actual content (text, images, etc.)
  2. Padding: Space inside the border, around content
  3. Border: A border around padding and content
  4. Margin: Space outside the border

Key Insight: Background color fills content + padding, but NOT margin!

3 / 25

Box Model CSS Properties

Setting Box Model Values:

.my-box {
  /* Content dimensions */
  width: 300px;
  height: 200px;

  /* Padding: space inside */
  padding: 20px; /* All sides */
  padding: 10px 20px; /* Top/bottom, left/right */
  padding: 10px 15px 20px 25px; /* Top, right, bottom, left */

  /* Border */
  border: 2px solid #667eea;

  /* Margin: space outside */
  margin: 30px auto; /* Centers horizontally */
}

The Box-Sizing Property (Game Changer!):

/* Default: width doesn't include padding/border */
* {
  box-sizing: content-box; /* Default - confusing! */
}

/* Better: width includes padding/border */
* {
  box-sizing: border-box; /* Use this! */
}

Pro Tip: Always start your CSS with * { box-sizing: border-box; }

This makes width calculations intuitive and predictable!

Box-Sizing: Content-Box vs Border-Box

Why This Property Changes Everything

❌ content-box (default)

width: 200px; padding: 20px; border: 10px;
Actual width = 260px!
200 + (20×2) + (10×2) = 260px
⚠️ Element is bigger than expected!

Two 50% columns with padding

content-box → total > 100%, so they wrap.

Column A (50% + padding + border)
Column B (50% + padding + border)

✓ border-box

width: 200px; padding: 20px; border: 10px;
Actual width = 200px!
Everything fits within 200px
✓ Predictable & intuitive!

Two 50% columns with padding

border-box → both fit side-by-side.

Column A (padding/border inside 50%)
Column B (padding/border inside 50%)

The Golden Rule - Always Use This:

/* Put this at the very top of your CSS file! */
*,
*::before,
*::after {
  box-sizing: border-box;
}

Why This Matters: Without border-box, creating layouts becomes a math nightmare!

Two 50% width columns with padding won't fit side by side with content-box.

The Display Property

Controlling How Elements Behave

Common Display Values:

/* Takes full width, stacks vertically */
.block { display: block; }

/* Flows with text, no width/height */
.inline { display: inline; }

/* Best of both worlds */
.inline-block { display: inline-block; }

/* Hide element completely */
.hidden { display: none; }

/* Modern layout methods */
.flex-container { display: flex; }
.grid-container { display: grid; }

Default Display Values:

  • Block elements: <div>, <p>, <h1>, <section>
  • Inline elements: <span>, <a>, <strong>, <em>

Remember: You can change ANY element's display property!

Turn a <span> into a block, or a <div> into inline-block

Flexbox: Modern 1D Layout

Flexible Box Layout

Flexbox makes it easy to align and distribute space among items in a container.

Item 1
Item 2
Item 3

Basic Flexbox Setup:

/* Parent container */
.container {
  display: flex;
  justify-content: space-between; /* Horizontal spacing */
  align-items: center; /* Vertical alignment */
  gap: 20px; /* Space between items */
}

/* Children automatically become flex items */
.item {
  flex: 1; /* Equal width */
}

Key Flexbox Properties:

  • flex-direction: row | column
  • justify-content: flex-start | center | space-between | space-around
  • align-items: stretch | center | flex-start | flex-end
  • flex-wrap: nowrap | wrap

CSS Grid: 2D Layout System

The Most Powerful Layout Tool

Grid allows you to create complex layouts with rows AND columns.

1
2
3
4
5

Basic Grid Setup:

.grid-container {
  display: grid;
  grid-template-columns: repeat(3, 1fr); /* 3 equal columns */
  grid-template-rows: 200px 200px; /* 2 rows of 200px */
  gap: 20px; /* Space between grid items */
}

Key Grid Properties:

  • grid-template-columns: Define column sizes
  • grid-template-rows: Define row sizes
  • gap: Space between grid items
  • grid-template-areas: Name grid areas for placement

Grid vs Flexbox: Use Grid for 2D layouts, Flexbox for 1D layouts

Typography in CSS

Making Text Beautiful & Readable

Font Properties:

body {
  /* Font stack with fallbacks */
  font-family: 'Helvetica Neue', Arial, sans-serif;
  font-size: 16px; /* Base size */
  line-height: 1.6; /* Spacing between lines */
  font-weight: 400; /* Normal weight */
}

h1 {
  font-size: 2.5rem; /* Relative to root */
  font-weight: 700; /* Bold */
  letter-spacing: -0.02em;
  text-transform: uppercase;
}

Using Google Fonts:

<!-- In your HTML head -->
<link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;700&display=swap" rel="stylesheet">

/* In your CSS */
body {
  font-family: 'Inter', sans-serif;
}

Font Size Units:

  • px: Absolute pixels
  • em: Relative to parent element
  • rem: Relative to root element (recommended!)
  • %: Percentage of parent

Color & Accessibility

Designing for Everyone

WCAG Color Contrast Requirements:

  • Normal text: 4.5:1 contrast ratio minimum
  • Large text (18px+): 3:1 contrast ratio minimum
  • AAA compliance: 7:1 for normal text
✓ Good Contrast (7:1)
White text on dark background is easy to read
✗ Poor Contrast (1.4:1)
Light yellow on yellow is hard to read

Testing Tools:

  • Chrome DevTools: Built-in contrast checker
  • WebAIM Contrast Checker: webaim.org/resources/contrastchecker
  • Browser extensions: axe DevTools, WAVE

Color Blind Considerations:

/* Don't rely on color alone */
.error {
  color: #d32f2f;
  border: 2px solid #d32f2f;
  background: #ffebee;
}

.error::before {
  content: "⚠ Error: "; /* Add text/icon indicator */

Responsive Design Fundamentals

One Site, All Devices

Core Principles:

  1. Fluid Grids: Use percentages, not fixed pixels
  2. Flexible Images: Images that scale with container
  3. Media Queries: Different styles for different screens

The Viewport Meta Tag (Essential!):

<!-- Add to HTML head -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">

Mobile-First Approach:

/* Base styles for mobile */
.container {
  width: 100%;
  padding: 20px;
}

/* Tablet and up */
@media (min-width: 768px) {
  .container {
    max-width: 750px;
    margin: 0 auto;
  }
}

/* Desktop and up */
@media (min-width: 1024px) {
  .container {
    max-width: 1200px;
  }
}

Responsive Units

Building Flexible Layouts

Relative Units for Responsive Design:

.responsive-text {
  /* Viewport units */
  font-size: 5vw; /* 5% of viewport width */
  padding: 2vh 4vw; /* Viewport height & width */

  /* Clamp for min/max limits */
  font-size: clamp(1rem, 4vw, 3rem); /* Min, preferred, max */
}

.flexible-container {
  /* Percentage of parent */
  width: 90%;
  max-width: 1200px;
  margin: 0 auto;

  /* Flexible spacing */
  padding: min(5vh, 50px);
}

Unit Reference:

  • vw: 1% of viewport width
  • vh: 1% of viewport height
  • vmin: 1% of viewport's smaller dimension
  • vmax: 1% of viewport's larger dimension
  • %: Percentage of parent element

Pro Tip: Use clamp() for responsive typography that doesn't get too small or too large!

CSS Custom Properties (Variables)

Write Once, Use Everywhere

Defining and Using Variables:

:root {
  /* Define variables in :root */
  --primary-color: #667eea;
  --secondary-color: #764ba2;
  --text-color: #333;
  --spacing: 20px;
  --border-radius: 10px;
}

/* Use variables with var() */
.button {
  background: var(--primary-color);
  color: white;
  padding: var(--spacing);
  border-radius: var(--border-radius);
}

.button:hover {
  background: var(--secondary-color);
}

Why Use Variables?

  • Consistent colors across your site
  • Easy theme switching
  • Change once, updates everywhere
  • More maintainable code

Best Practice: Define all colors, spacing, and fonts as variables!

CSS Best Practices

Writing Maintainable CSS

1. Use Meaningful Class Names:

/* Bad: Generic, unclear */
.box1 { }
.red-text { }
.big { }

/* Good: Descriptive, purposeful */
.navigation-menu { }
.error-message { }
.hero-title { }

2. Organize Your CSS:

/* 1. Reset/Normalize */
* { margin: 0; padding: 0; box-sizing: border-box; }

/* 2. Variables */
:root { --primary-color: #667eea; }

/* 3. Base Styles */
body { font-family: Arial; }

/* 4. Layout */
.container { max-width: 1200px; }

/* 5. Components */
.button { padding: 10px 20px; }

/* 6. Utilities */
.text-center { text-align: center; }

3. Performance Tips:

  • Avoid overly specific selectors
  • Minimize use of !important
  • Group similar styles
  • Use shorthand properties when possible
19 / 26

Common CSS Mistakes to Avoid

1. Forgetting Box-Sizing:

/* Always include this! */
* {
  box-sizing: border-box;
}

2. Using IDs for Styling:

/* Avoid: Too specific */
#header { background: blue; }

/* Better: Use classes */
.header { background: blue; }

3. Not Testing Responsive Design:

  • Always test on multiple screen sizes
  • Don't assume desktop-only
  • Mobile-first is easier to scale up

4. Overusing !important:

/* Bad: Creates specificity wars */
.text { color: red !important; }

/* Better: Fix specificity issues properly */
.container .text { color: red; }

5. Not Considering Accessibility:

  • Always check color contrast
  • Don't remove focus outlines without replacement
  • Test with keyboard navigation

Debugging CSS Like a Pro

Tools and Techniques

1. Browser DevTools (F12):

  • Elements tab: Inspect and modify styles live
  • Computed tab: See final calculated values
  • Coverage tab: Find unused CSS
  • Device emulation: Test responsive design

2. Debug Techniques:

/* Temporary debug borders */
* {
  border: 1px solid red;
}

/* Check element bounds */
.debug {
  background: rgba(255,0,0,0.1);
  outline: 2px dashed blue;
}

3. Common Issues & Solutions:

  • Styles not applying: Check specificity and cascade
  • Layout breaking: Check overflow and box-sizing
  • Unexpected spacing: Check margin collapse
  • Z-index not working: Element needs position (not static)

Pro Tip: Right-click → Inspect Element is your best friend!
